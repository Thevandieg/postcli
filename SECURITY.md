# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch state.

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Instead:

1. Share a private report with maintainers, including:
   - A clear description of the issue
   - Steps to reproduce
   - Impact assessment
   - Any suggested mitigation
2. If available, use GitHub's private vulnerability reporting for this
   repository.

You can also open an issue with minimal details asking for a private contact
channel if private reporting is not yet enabled.

## What to Expect

- We will acknowledge receipt as soon as practical.
- We will investigate and validate the report.
- We will coordinate a fix and release timeline based on severity.
- We may request additional details to reproduce or verify impact.

We appreciate responsible disclosure and efforts to keep users safe.
